Comment on dimensionality and distribution of classes

The classes' distribution can affect the classification, since if the prior for class A is a lot greater than prior of the other classes, the product likelihood*prior will be also greater (supposing the likelihood is not varying too much from one class to another) and will "attract" more data.

Focusing on the dimensionality, the highest the dimensionality, the highest the number of combination of the features. If there are too many features but not enough data, it is not possible to have a sufficient number of data for each possible combination of features. This results in overfitting: we are creating a model using a set of data that describes only a part of the total dataset, that correspond to not being able to caracterize all the dataset corresponding to all the possible features. Below there is a plot showing the distribution of the elements per class: in the MNIST dataset it is possible to notice a greater variance in the distribution.

Data Cleaning

Before operating on the data, it could be useful to identify eventual outliers. Proceding in this way can possibly lead to a better solution later on. The procedure is the following:
  • for each class calculate the mean
  • calculate the average distance and the standard deviation of the distances for each class
  • eliminate the images which distance from the mean is not in the interval [average_distance - standard_dev,average_distance + standard_dev]

Nave Bayes

Pros and Cons


It is possible to identify pros and cons of using this kind of classifier (Naive Bayes). A NB classifier is a simple classifier, and it is also very fast to converge if the assumption of independence holds. If it doesn't hold this method still allows to arrive to a good solution, but not the best one. On the other hand, if data are skewed, that means more training examples for one class than another, it can cause the decision boundaries to be biased. This cause the classifier to unwillingly prefer one class over the other. A similar trend can be seen when the indipendence costraint is not respected: the likelihood tend to be greater for classes that most violate the indipendence assumption.

Our Classifier


Our classifier consist in a python class able to calculate the mean and variance of the training data, in order to evaluate the likelihood of the new data. In particular we used the probability density function in order to calculate the "probability", and not the actual probability (the integral of the function). This because with the actual probability, the highest the value with respect to the mean, the highest is going to be the probability. In our case we want the values closest to the mean to have more impact, that's why the use of the pdf.

When the likelihood is computed, since we are implementing a naive method, we can consider the different probabilities of the different features as indipendent, and simply multiply them with one another. A common problem is to end up with a very small number at the end, since we are multiplying numbers that are less than 1.
A solution could be to use the logarithm, and since the logarithm of a multiplication is the sum of the log of each element, that is what is computed instead of the multiplication. One fact to keep in mind is the following: the log of a probability is less than 0, since the probability is less than 1. When we calculate the likelihood we obtain something less than 0 that is going to be divided by something that is also less than zero (sum over all the classes of likelihood*prior). The numerator is going to be smaller and smaller as the likelihood is moving from a value of 1 to 0: so we obtain something less than 0 and very small. When this result is divided by something also less than 0 is going to be a big positive number.

What we obtain?


The smaller the likelihood, the smaller the log of it, but at the end is going to be very big. At the end we are not looking for the max value, we are rather looking for the smallest one thanks to this inversion of sign.

Log-Likelihood tuning


Sometimes it is possible that the log of the likelihood is -inf: this is due to a too small value of the likelihood. A part of data cleaning could be getting rid of -inf values, that during the sum of the logs will wrongly influenciate the result.

class NaiveBayes:
    def __init__(self,classes,training_data,training_labels,test_data,test_labels):
        self.classes = classes
        self.training_data = training_data
        self.training_labels = training_labels
        self.test_data = test_data
        self.test_labels = test_labels
        self.mu = []
        self.sigma2 = []
        self.priors = np.array([])
        self.predictions = np.array([])
    
    def mean_and_var(self):
        for c in self.classes:
            imgs = self.training_data[self.training_labels == c]
            var = np.std(imgs,axis=0, ddof=1)
            mn = np.mean(imgs,axis=0)
            self.mu.append(mn)
            self.sigma2.append(var)
        self.mu = np.array(self.mu)
        self.sigma2 = np.array(self.sigma2)

        
    def gaussian(self, x, m, v):
        return 1 / np.sqrt(2 * np.pi * v) * np.exp(-(x - m)**2 / (2 * v))
    
    def likelihood(self, cls, data):
        """Used to calculate the likelihood"""
        
        log_prob = np.array([np.log(self.gaussian(d, m, v)) if v > 0.0 else 0 for 
                         m, v, d in zip(self.mu[cls], self.sigma2[cls], data)])
        mn = min(log_prob[log_prob!=-math.inf])
        return sum(                
                  [d if d != -math.inf else mn-1 for d in log_prob]  
               )
    
    def prior(self):
        """Used to calculate the prior"""
        
        pr = []
        for c in self.classes:
            mask = self.training_labels == c
            pr.append(len(self.training_data[mask]))
        self.priors =  np.array([e/sum(pr) for e in pr])
        
    
    def predict(self):
        """Used to predict the labels of the test dataset"""
        
        labels = []
        for d_test_img, d_test_lbl in zip(self.test_data,self.test_labels):            
            probs = []
            for c in self.classes:           
                probs.append(self.likelihood(c,d_test_img)*self.priors[c])                
            s = np.sum(probs)
            probs = [p/s for p in probs]
            labels.append(probs.index(min(probs)))
        self.predictions = np.array(labels)        
                
    
    def accuracy(self):
        """Calculate the accuracy
        
            Returns:
              - number of correct predictions
              - number of incorrect predictions              
              - accuracy in %
        """
        
        err = ok = 0
        for pred, eff in zip(self.predictions, self.test_labels):
            if pred == eff:
                ok = ok + 1
            else:
                err = err + 1
        acc = ok/(ok+err)
        return (ok,err,acc)
    
    def save_predictions(self, file_name):
        f = open(file_name,'wb')
        pickle.dump(self.predictions,f)
        f.close()
        
    def load_predictions(self, file_name):
        f = open(file_name,'rb')
        self.predictions = pickle.load(f)
        f.close()

def plot_confusion_matrix(ax, true_label, prediction, classes, 
                         title='Confusion matrix', cmap=plt.cm.Blues,
                         show=True):
    
    cm = confusion_matrix(true_label, prediction)
    np.set_printoptions(precision=2)

    ax.imshow(cm, interpolation='nearest', cmap=cmap)
    ax.set_title(title)
    tick_marks = np.arange(len(classes))
    ax.set_xticks(tick_marks, classes)
    ax.set_yticks(tick_marks, classes)
       
    thresh = cm.max() / 2.
    for i, j in itertools.product(range(cm.shape[0]), range(cm.shape[1])):
        ax.text(j, i, math.ceil(cm[i, j]*10000)/10000,
                 horizontalalignment="center",
                 color="white" if cm[i, j] > thresh else "black")

    
    ax.set_ylabel('True label')
    ax.set_xlabel('Predicted label')
    
    if show == True:
        plt.show()

Bayesian Linear Regression

The class below is used to create the bayesian linear regression method that will allow us to predict the type of an image using linear regression. Before being able to operate on the data, it is necessary to modify their structure, without changin the meaning and the content of each image. We defined the prior over w (p(w)) as a diagonal matrix with the element's value equal to 10000.

class BLR():
    def __init__(self, classes, training_data, training_labels, test_data,test_labels, sigma, binary=True):     
        """ Parameters:
             - classes: set with the classes for the classification
             - sigma: the sigma used for the prior over w
             - binary: if True one-hot vector encoding is used
        """
        
        # rapresent each class as a one-hot vector if binary = True
        self.classes = np.eye(len(classes)) if binary == True else classes
        self.training_data = training_data
        # rapresent each label as a one-hot vector
        self.training_labels = [self.classes[c] for c in training_labels]\
                        if binary == True else training_labels
        self.test_data = test_data
        self.test_labels = test_labels
        self.sigma = sigma   
        self.binary = binary
        
    def adjust_data(self, k):
        """modify the input data to have them in the right format"""
        
        self.k = k
        def redifine_data(x):
            return np.array([
                    np.array([x_i**n for n in range(k + 1)]).flatten() for x_i in np.array(x)
                ])
        X = redifine_data(self.training_data)
        self.training_data = np.hstack( (np.array([np.ones(len(X))]).T, X ))        
        X = redifine_data(self.test_data)
        self.test_data = np.hstack( (np.array([np.ones(len(X))]).T, X ))
    
    def generate_w(self):    
        """Used to generate w given training data"""
        
        X = self.training_data
        t0 = time.time()
        X2 = X.T.dot(X)/self.sigma
        S = np.eye(len(X2))*10000
        X2inv = np.linalg.inv(X2+np.linalg.inv(S))
        self.w = X2inv.dot(X.T).dot(self.training_labels)/self.sigma
        
    def make_prediction(self):
        """Makes prediction given test data and w"""
        
        self.predictions = self.test_data.dot(self.w)
        if self.binary == True:
            self.predictions = [np.argmax(e) for e in self.predictions]
            
    def change_test_set(self, test_data, test_labels):
        """Allow to change the test set, in order to test the model with different dataset"""
        
        self.test_data = np.array([
                    np.array([x_i**n for n in range(self.k + 1)]).flatten() for x_i in np.array(test_data)
                ])
        
        self.test_labels = test_labels
      
    def accuracy(self):
        """Calculate the accuracy
        
            Returns:
              - number of correct predictions
              - number of incorrect predictions              
              - accuracy in %
        """
        err = ok = 0
        for pred, eff in zip(self.predictions, self.test_labels):
            if round(pred) == eff:
                ok = ok + 1
            else:
                err = err + 1
        acc = ok/(ok+err)
        return (ok,err,acc)
    
    def save_model(self, file_name):
        """Save the model, that means saving w and the parameters sigma, binary, K, classes"""
        
        f = open(file_name+'_w','wb')
        pickle.dump(self.w,f)
        f.close()
        f = open(file_name+'_param','wb')
        pickle.dump([self.sigma,self.binary,self.k,self.classes],f)
        f.close()
    
    def load_model(self, file_name):
        """Load the model, that means loading w and the parameters sigma, binary, K, classes"""
        
        f = open(file_name+'_w','rb')
        self.w = pickle.load(f)
        f.close()
        f = open(file_name+'_param','rb')
        [self.sigma,self.binary,self.k,self.classes] = pickle.load(f)
        f.close()

def MSE(predicted_labels, target_labels):
    error = 0
    for p, t in zip(predicted_labels, target_labels):
        error += np.square(p-t)
    return error/len(predicted_labels)

Problems encountered and solution proposed


We are working with discrete variables, but linear regression works with continues variables. In order to solve this problem we decided to treat each label as a one-hot vector. In this way we will not obtain a vector of weights of dimension K+1 (where K is the grade of the polynomial and the + 1 is the grade zero term), but a matrix of dimension K+1 x number_different_classes. At the end of the prediction phase, for each test image not a single class will be returned but a set of N predictinos, where N is the number of different classes. Picking the highest value will give us the (hopefully) correct prediction.

We are going to evaluate this method using:
  • MNIST dataset
  • MNIST dataset without outliers
  • CFAR dataset

Limitation on the previous approach


While using the one-hot vector to define a class for a given image, we are saying that the "contribute" of that image for a given class C is maximum, but completely inexistent for all the other classes. It could be better to procede in the following way:
  • given a class, take the images that are far away from the mean
  • for that images look if they can contribute in defining also other classes (not a one-hot vector anymore)
In order to exploit this idea we created a class, that behaves exactly as BLS, but with the addition of the methods to build up the labels to use in the training.

Bayesian Linear Regression - new approach

We are starting to calculate the average image for each class and then the distances of the images belonging to each class with the average calculated before.
Having now the set of distances for each class, we evaluate mean and variance. Supposing that the distribution of the distances follows a normal distribution, mean and variance are used to evaluate the pdf for each class. Given a distance, if the evaluation of the pdf in that point is small enough with respect to the maximum of the pdf it means that the image is far away from the class mean, so it is possible to look how similar it is to the other classes.
Operating in this way lead us to the following situation:
  • given a class, some images have as label a one-hot vector
  • given the same class, other images will have as label a vector with: a 1 with respect to their class and other numbers with respect to other classes.

In order to identify the images far enough from the class mean, we are looking at the ratio between the pdf evaluated for a given distance and the max of the pdf: if 1 - ratio is greater than 0.8, then the image is considered far from the mean.

How to fill the label vector


We said before that when an image is far from its class mean, the label will be a vector with a 1 with respect to their class and other numbers with respect to other classes. Given an image I of class C1, the vector-label will have 1 with respect to C1 and pdf(distance_between_I_and_mean_of_C) for each class C different from C1.

class BLR_mod(BLR):
    def __init__(self, classes, training_data,
                 training_labels, test_data,test_labels, sigma, binary=True):
        self.original_classes = classes
        self.classes = np.eye(len(classes)) if binary == True else classes
        self.training_data = training_data
        self.training_labels = training_labels
        self.test_data = test_data
        self.test_labels = test_labels
        self.mu = []
        self.sigma2 = []
        self.top_gaussian = []
        self.class_mean = []
        self.sigma = sigma   
        self.binary = binary
        
    def gaussian(self, x, m, v):
        return 1 / np.sqrt(2 * np.pi * v) * np.exp(-(x - m)**2 / (2 * v))
    
    def mean_and_var(self):
        """Function that calculates mean and variance of the distances for each class"""
        
        for c in self.original_classes:
            class_mean = np.mean(self.training_data[self.training_labels == c],axis=0)
            distances = [np.linalg.norm(class_mean-e) for e in self.training_data[self.training_labels == c]]
            # var of the distances
            var = np.std(distances,axis=0, ddof=1)
            # mean of the distances
            mn = np.mean(distances,axis=0)
            self.class_mean.append(class_mean)
            self.top_gaussian.append(self.gaussian(mn,mn,var))
            self.mu.append(mn)
            self.sigma2.append(var)
        self.mu = np.array(self.mu)
        self.sigma2 = np.array(self.sigma2)

    
    def labels(self):
        """Used to redifine the label vectors, in order to create them not as one-hot vector"""
        
        labels = np.zeros((len(self.training_labels),10))
        i = 0
        for c,e in zip(self.training_labels,self.training_data):
            dist = np.linalg.norm(self.class_mean[c]-e)
            ratio = 1 - self.gaussian(dist,self.mu[c],self.sigma2[c])/self.top_gaussian[c]
            labels[i] = self.classes[c]
            if ratio > 0.8:
                for cls in self.original_classes:
                    if cls != c:
                        dist = np.linalg.norm(self.class_mean[cls]-e)
                        labels[i][cls] = self.gaussian(dist,self.mu[cls],self.sigma2[cls])
            i += 1
        self.training_labels = labels
We are just defining an utility function in order not to repeate the code many times. It allows to decide when to do the training or just load the model.

Naive bayesian vs Bayesian Linear Regression

The Naive Bayesian classifier was performing quite good, especially with respect to bayesian linear regression. The fact that the bayesian linear regression was performing really bad was due to the usage of discrete classes as continuous ones: this problem was solved introducing the one-hot vector rapresentation.
With the one-hot vector rapresentation was like building a different model for each class, taking into account only the images belonging to a specific class and not all of them. This solution was way better than the one obtained with the naive bayesian classification.
The dicision to take into account also the distance between an image of class C and all the other classes different from C allowed to improve even more the solution.

It is necessary to notice that we inserted some semplification in the data rapresentation for the bayesian linear regression part. We rapresented the first column of the matrix containing all the images as a column of ones. In reality it should not have been a single column, but a set of 784 column all set to 1. Using this approach the result obtained will be a bit better than the one obtained now. The side effect though would have been a huge amount of time in order to finish the computation of X.T.dot(X). We decided not to use it and stick with the more simple data format.
